# Security Review

`flow-design`、`flow-review`、`flow-verify` 都要考虑安全，但深度不同：

- design：识别安全边界和威胁。
- review：检查方案是否有明显漏洞。
- verify：检查实现是否符合方案和常见安全要求。

## 检查项

- 认证：未登录、过期 token、会话状态。
- 授权：角色、资源归属、租户隔离、越权访问。
- 输入：参数校验、类型、长度、枚举、文件上传、富文本。
- 输出：敏感字段、错误信息、日志、前端暴露。
- 数据：事务、一致性、删除/恢复、审计。
- API：幂等性、重放、限流、CORS、CSRF。
- 依赖：新增依赖、脚本执行、供应链风险。
- 前端：XSS、危险 HTML、URL 跳转、localStorage 敏感数据。

## 输出要求

安全审查必须给出：

```text
结论：PASS / BLOCKED / NEEDS-FIX
高风险问题：
中风险问题：
可接受风险：
必须验证的安全用例：
```
