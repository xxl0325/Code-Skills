# Verification Rules

完成不等于写完代码。必须有证据证明目标达成。

## 验证层级

1. 静态检查：lint、typecheck、format、架构规则。
2. 单元/集成测试：覆盖核心逻辑和失败路径。
3. 构建验证：前后端构建或打包通过。
4. 运行时验证：服务启动、接口请求、浏览器页面、关键用户流。
5. 安全验证：认证授权、输入校验、敏感数据、依赖风险、日志泄露。

## 记录格式

每个验证项记录：

```text
- 命令/动作：
- 结果：
- 证据：
- 失败时的修复或后续：
```

## 失败路由

- 实现偏差或测试失败：回 `flow-build`。
- 计划遗漏或验收标准错误：回 `flow-plan`。
- 技术方案错误：回 `flow-design`，再走 `flow-review`。
- 需求理解错误：回 `flow-discuss`。

## 不允许的完成声明

- “看起来没问题”
- “应该可以”
- “未运行但理论上通过”
- “只改了很小，不需要测”

如果无法运行验证，必须说明原因、影响和替代证据。
